**Техническое задание на проект: NBA PER Analyzer**



1\. Цель проекта

Разработка веб-сервиса для автоматизированного сбора спортивной статистики и расчета продвинутых метрик эффективности баскетболистов NBA.

Проблема, которую решает сайт: базовая статистика (очки, подборы, передачи) не дает объективной картины полезности игрока на площадке. Ручной сбор данных из разных источников и расчет комплексных математических формул (таких как PER — Player Efficiency Rating) занимает у аналитиков много времени.



*Сервис помогает пользователям:*

* Быстро находить нужных игроков и загружать их актуальную статистику через внешние источники.
* Автоматически рассчитывать рейтинг PER на основе сложных алгоритмов с использованием инструментов анализа данных.
* Визуально сравнивать результативность игрока со средними показателями по лиге для скаутинга и составления отчетов.



2\. Роли пользователей

1. Гость - посетитель, который может просматривать главную страницу с описанием методологии расчета PER, а также имеет доступ к общему справочнику команд лиги. Имеет возможность зарегистрироваться.
2. Спортивный аналитик (авторизованный пользователь) - основной пользователь. Имеет доступ к расширенному поиску, может инициировать загрузку свежей статистики по конкретным игрокам, формировать личный "список отслеживания" и просматривать индивидуальные графики эффективности.
3. Администратор - суперпользователь, имеющий доступ к панели Django Admin. Управляет справочниками команд и игроков, может принудительно очищать кэш статистики и управлять правами пользователей.



3\. Модели данных

*Модель 1: Team (Команда)*

Справочник баскетбольных франшиз, необходимый для группировки игроков.

* Поля: name (название, например - Golden State Warriors), city (город, например - San Francisco), abbreviation (аббревиатура, например - GSW), conference (конференция: Запад/Восток).



*Модель 2: Player (Игрок)*

Карточка конкретного баскетболиста.

* Поля: team (связь с Team, ForeignKey), first\_name (имя), last\_name (фамилия), position (игровая позиция), api\_id (уникальный идентификатор игрока во внешнем API для привязки данных).



*Модель 3: SeasonStat (статистика за сезон)*

Агрегированные статистические данные игрока за конкретный игровой год, необходимые для математических расчетов.

* Поля: player (связь с Player, ForeignKey), season\_year (год сезона), games\_played (количество игр), minutes (сыгранные минуты), points (очки), assists (передачи), rebounds (подборы), steals (перехваты), blocks (блок-шоты), turnovers (потери).



*Логика:* модель содержит метод для вызова расчетного скрипта, который вычисляет и сохраняет итоговое значение в поле calculated\_per.



4\. Ключевой функционал

*Формирование базы данных:* 

* При поиске игрока система проверяет локальную базу. Если актуальной статистики нет, сервис автоматически обращается к внешнему API, загружает данные за выбранный сезон, парсит их и сохраняет в БД для дальнейшего использования.

*Продвинутая аналитика:* 

* При запросе отчета бэкенд передает сырые статистические метрики игрока в аналитический модуль. Модуль нормализует данные и применяет математическую формулу Холлинджера для вычисления единого индекса эффективности.

*Визуализация профиля игрока:* 

* На детальной странице баскетболиста строится радарная диаграмма или столбчатый график, визуально демонстрирующий сильные и слабые стороны игрока (атака, защита, плеймейкинг) в сравнении со средними показателями по его амплуа.



5\. Внешние интеграции / Аналитика

*Внешнее API:* 

* Интеграция с публичным баскетбольным API (например, balldontlie API или открытыми эндпоинтами NBA) через библиотеку requests для легального получения реальных игровых данных в формате JSON.

*Библиотека Pandas:* 

* Использование библиотеки pandas на бэкенде для структурирования загруженных массивов данных в DataFrame, их очистки, фильтрации выбросов и векторизованного вычисления итогового рейтинга эффективности.

*Визуализация:* 

* Генерация статических графиков сравнения метрик с помощью matplotlib (в формате PNG/Base64 для встраивания в шаблон).
