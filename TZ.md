**Техническое задание на проект: NBA PER Analyzer**



1\. Цель проекта - разработка веб-сервиса на базе Django для автоматизированного сбора спортивной статистики и расчета продвинутых метрик эффективности баскетболистов NBA.

Проблема, которую решает сервис - базовая изолированная статистика (только очки или только передачи) не дает объективной картины полезности игрока на площадке. Ручной сбор данных и расчет комплексных формул занимает много времени. Сервис автоматизирует этот процесс, вычисляя динамический рейтинг эффективности (Simplified PER) для каждого игрока.



*Сервис помогает:*



* Пакетно загружать актуальную статистику игроков через внешние источники.
* Автоматически рассчитывать рейтинг PER на основе расширенного бокс-скора.
* Структурировать данные по конференциям и командам для удобного скаутинга.



2\. Роли пользователей



* Гость — имеет доступ к просмотру общего списка команд лиги и детальной странице каждой команды со списком игроков и их статистикой.
* Администратор — имеет доступ к панели Django Admin. Управляет справочниками команд и игроков, контролирует целостность данных.



3\. Модели данных



*Модель 1: Team (Команда)*



Справочник баскетбольных франшиз для группировки игроков.

* `abbreviation` — трехбуквенный код (например, GSW, LAL).
* `city` — город (например, San Francisco).
* `name` — название (например, Warriors).
* `conference` — конференция (East/West).



*Модель 2: Player (Игрок)*



Карточка конкретного баскетболиста.

* `team` — привязка к команде.
* `first\_name` — имя.
* `last\_name` — фамилия.
* `position` — игровая позиция (PG, SG, SF, PF, C).
* `api\_id` — идентификатор для синхронизации.



*Модель 3: SeasonStat (Статистика за сезон)*



Агрегированные показатели игрока за конкретный игровой год, расширенные для вычисления формулы PER.

* `player` — привязка к игроку.
* `season\_year` — год сезона.
* `games\_played` — сыгранные матчи.
* `minutes` — минуты на паркете.
* `points` — набранные очки.
* `rebounds` — подборы.
* `assists` — результативные передачи.
* `steals` — перехваты.
* `blocks` — блок-шоты.
* `turnovers` — потери.
* `fgm` / `fga` / `fg\_pct` — попадания, попытки и процент с игры.
* `three\_pm` / `three\_pa` / `three\_pct` — попадания, попытки и процент трехочковых.
* `ftm` / `fta` / `ft\_pct` — попадания, попытки и процент штрафных бросков.



Логика модели:

* Класс содержит динамическое свойство (property) для расчета индекса Simplified PER игрока в реальном времени на основе разницы положительных и отрицательных технико-тактических действий.



4\. Ключевой функционал



*Формирование и очистка базы данных*



Реализована кастомная консольная команда управления Django (`load\_nba\_data`). При её вызове система:

* Производит полную каскадную зачистку старых таблиц для исключения дублирования данных.
* Инициализирует актуальную структуру из 30 современных команд NBA.
* Осуществляет пакетный сбор данных, отсеивая игроков без игровой практики.



*Продвинутая аналитика*

* Расчет рейтинга PER происходит на лету без избыточного сохранения вычисляемых значений в СУБД. Формула учитывает очки, подборы, передачи, перехваты, блоки, а также штрафует за потери мяча и нереализованные броски с игры и штрафной линии.



*Отображение данных*

* На детальной странице команды разворачивается полная интерактивная таблица (бокс-скор), где ключевые аналитические метрики визуально сгруппированы для быстрого анализа состава.



5\. Внешние интеграции и инструменты анализа



*Модуль requests:* 

* Используется для выполнения сетевых HTTP-запросов к удаленным серверам и извлечения сырых массивов данных.



*Библиотека Pandas:* 

* Применяется на бэкенде в скрипте загрузки для структурирования данных в DataFrame, векторизованной очистки пустых значений, фильтрации артефактов и корректного разбиения имен игроков.
