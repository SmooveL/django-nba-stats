# **NBA Stats Analyzer**



Веб-сервис для автоматизированного сбора, очистки и визуализации спортивной статистики баскетболистов NBA. Проект решает проблему нестабильности публичных спортивных API, агрегируя исторические данные и предоставляя продвинутую аналитику (индекс Simplified PER) без задержек и лимитов на количество запросов.



**Ссылка на рабочий проект:** [http://51.250.102.66](http://51.250.102.66)



### **Архитектура и ключевые решения**

Проект спроектирован с упором на надежность, консистентность данных и оптимизацию запросов.

1\. Пакетная обработка: 

* Вместо нестабильного Real-time API реализована кастомная команда управления Django. Она скачивает сырые CSV-данные, проводит их очистку и трансформацию с помощью Pandas (обработка NaN, нормализация названий команд) и загружает в СУБД.

2\. Динамические вычисления: 

* Индекс эффективности (Simplified PER) не хранится статично в базе данных. Он вычисляется «на лету» в оперативной памяти через декоратор `@property` в моделях, что исключает дублирование данных и рассинхронизацию.

3\. In-Memory генерация графиков:

* Радарные диаграммы профиля игрока генерируются с помощью Matplotlib и NumPy напрямую в байтовый поток и кодируются в `base64`. Сервер не сохраняет временные .png файлы на жесткий диск, что экономит память и ускоряет рендеринг.



### **Стек технологий**

* Backend: Python 3.x, Django 6.x
* СУБД: PostgreSQL, psycopg2-binary
* Анализ данных и визуализация: Pandas, Matplotlib, NumPy
* Frontend: HTML5, Bootstrap 5 (наследование шаблонов, адаптивная сетка)

### Как запустить проект локально

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/SmooveL/django-nba-stats.git
   cd django-nba-stats

2. **Создайте и активируйте виртуальное окружение:**
    ```bash
   python -m venv venv
    source venv/bin/activate  # для Linux/Mac
    venv\Scripts\activate     # для Windows
3. **Установите зависимости:**
    ```bash
   pip install -r requirements.txt

4. **Выполните миграции:**
    ```bash
   python manage.py migrate

5. **Запустите сервер:**
    ```bash
   python manage.py runserver

6. **Откройте проект в браузере:**
    ```
   Перейдите по ссылке: http://127.0.0.1:8000/
   
### Как развернуть на сервере 

Проект полностью контейнеризирован и готов к запуску на сервере через `docker-compose`.

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/SmooveL/django-nba-stats.git
   cd django-nba-stats

2. Создайте файл .env в корневой папке и укажите необходимые переменные окружения:
   ```bash
   SECRET_KEY=секретный_ключ
   DB_NAME=nba_db
   DB_USER=nba_user
   DB_PASSWORD=пароль
   DB_HOST=db
   DB_PORT=5432

3. Соберите и запустите контейнеры:
   ```bash
   docker compose up -d --build

4. Выполните миграции, соберите статику и запустите парсер для заполнения БД:
   ```bash
   docker compose exec web python manage.py migrate
   docker compose exec web python manage.py collectstatic --noinput
   docker compose exec web python manage.py load_nba_data
